# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Purpose

Integration test project for [premex-ab/setup-android](https://github.com/premex-ab/setup-android). Each `sdk-*` folder is a standalone minimal Android Gradle project that builds with a specific cmdline-tools version. CI runs all of them across Ubuntu, Windows, and macOS.

## Build Commands

```bash
# Build a specific SDK version
cd sdk-{N}
./gradlew --no-daemon assembleDebug

# Sync SDK versions (fetches new cmdline-tools from Google, creates folders, updates CI matrix and docs)
python3 scripts/sync-sdk-versions.py
```

## Architecture

- **`sdk-*` folders** — Each is an independent Gradle project with its own wrapper, AGP version, and Java compatibility. The app itself is a no-op `MainActivity` with zero dependencies.
- **`scripts/sync-sdk-versions.py`** — Parses Google's [repository2-3.xml](https://dl.google.com/android/repository/repository2-3.xml) to discover stable cmdline-tools versions, creates missing `sdk-*` folders from templates, and regenerates `.github/workflows/build.yml`, `README.md`, and `CLAUDE.md`.
- **`.github/workflows/build.yml`** — CI matrix: OS × SDK version. Auto-generated by the sync script — do not edit the matrix entries by hand.
- **`.github/workflows/sync-sdk-versions.yml`** — Weekly (Monday 06:00 UTC) cron job that runs the sync script and opens a PR if new versions are found. Uses a GitHub App token for authentication.
